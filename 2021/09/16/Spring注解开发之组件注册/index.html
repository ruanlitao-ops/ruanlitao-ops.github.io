二、添加包扫描

包扫描使用的是 @ComponentScan 注解，可以指定要扫描的组件的范围。

1. @ComponentScan 注解

此时在配置类中就可以使用 @ComponentScan 注解的 value 属性指定包扫描范围

2. excludeFilters 属性

可以使用 excludeFilters 属性配置排除规则，及在扫描组件时排除掉一些不添加到 ioc 容器中去，其值是一个 Filter 数组，通过 type 指定过滤的规则，可以是按注解、aspectJ 表达式、正则表达式来指定过滤的类型，classes 指定过滤的类。

详细讲一下过滤规则：

  • FilterType.ANNOTATION：按照注解方式，classes 中指定注解类，如 classes = {Controller.class, Repository.class}
  • FilterType.ASSIGNABLE_TYPE：按照指定的类型，classes 中指定具体类，如 classes = {PersonController.class}
  • FilterType.ASPECTJ：使用 ASPECTJ 表达式，不常用
  • FilterType.REGEX：使用正则表达式
  • FilterType.CUSTOM：使用自定义规则，classes 中指定自定义类型过滤类，该类要实现 TypeFilter 接口，并重写 match 方法，value 指定的包下的每一个类都会进入到 match 方法，只有返回值为 true 时，这个类才会符合过滤条件

使用 FilterType.ANNOTATION：这里就是排除了使用 @Controller 和 @Service 注解的组件加入 ioc 容器中，最后在容器中的只有 personDao

3. includeFilters 属性

可以使用 includeFilters 注解来指定扫描策略，其作用和 excludeFilters 相反，就是指定要将那些组件注册到 ioc 容器中去，使用方式和 excludeFilters 相似，都是配置 Filter，这里就不赘述了。

但是在使用 includeFilters 是，要将默认的扫描规则关闭掉，即设置 useDefaultFilters 属性为 false。

4. @ComponentScans 注解

@ComponentScan 是一个重复注解，可以使用多次，指定多个扫描规则，为了方便，也可以使用@ComponentScans， 该注解中可以指定多个包扫描规则。

三、@Scope 注解

@Scope 注解可以指定当前 bean 是多实例还是单实例，它源码中有这些值

  • ConfigurableBeanFactory.SCOPE_PROTOTYPE：多实例的，使用 @Scope(“prototype”) 指定
  • ConfigurableBeanFactory.SCOPE_SINGLETON：单实例的，默认值
  • org.springframework.web.context.WebApplicationContext.SCOPE_REQUEST：同一次请求创建一个实例，使用 @Scope(“request”) 指定
  • org.springframework.web.context.WebApplicationContext.SCOPE_SESSION：同一次 session 创建一个实例，使用 @Scope(“session”) 指定

指定默认单实例对象时，ioc 容器启动会调用方法创建对象放到 ioc 容器中，以后每次获取就是直接从容器中拿，就不会调用方法了。

指定多实例对象时，ioc 容器启动时不会调用方法创建对象放在容器中，是在每次获取 bean 时调用方法创建对象放入 ioc 容器中，每次获取都会调用方法创建对象。

懒加载：在容器启动时不创建对象，第一次获取 bean 时才创建对象，并将对象放入容器中，懒加载时针对单实例对象的。使用 @Lazy 注解

四、@Conditional 注解

@Conditional 注解：会按照一定的条件进行判断，满足条件才将 bean 注册到容器中，可以标注在方法上，也可以标注在配置类上，只有当条件符合的时候才注册 bean。

六、Spring 提供的 FactoryBean

Spring 提供了一个 FactoryBean 接口，可以注册组件到 ioc 容器中，其中有三个方法

  • getObject() 可以将返回值是泛型中指定的类型，并且可以将返回值注册到 ioc 容器中
  • getObjectType() 返回泛型中指定的类型的 class 对象
  • isSingleton() 当前组件是否是单例的，是单实例的返回 true，是多实例就返回 false，默认是 true，即单实例

七、总结在容器中组件注册方法

1. 开启包扫描

对于自己写的类，可以根据不同的功能在类上加上 @Component、@Controller、@Service、@Repository 等注解，开启包扫描后就可将这些类注册到 ioc 容器中。

但是对于不是自己写的类，就不能使用这种方法了。

2. @Bean 注解

在方法上使用 @Bean 注解，将自己或者第三方包里的组件注册到 ioc 容器

3. @Import 注解

快速给容器中导入一个组件，有三种导入方法：

  1. @Import(要导入到容器中的组件)；容器中就会自动注册这个组件，id 默认是全类名
  2. ImportSelector：返回需要导入的组件的全限定类名数组
  3. ImportBeanDefinitionRegistrar：手动注册 bean 到容器中

4. FactoryBean 接口

Spring 提供的工程 bean，写一个类实现 FactoryBean 接口，有方法可以注册组件。

5. @Conditional 注解

按照一定的条件进行判断，将满足条件的 bean 注册到容器中。
